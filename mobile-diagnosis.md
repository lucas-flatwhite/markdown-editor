# 모바일 문제 진단 결과

## 가용 폭 vs 필요 폭 (iPhone 375px 기준)

- 가용 폭: 343px (375 - 32px 패딩)
- 필요 폭: 553px (좌 279px + 우 274px)
- 초과: **210px 오버플로우**

## 문제 영역 상세

### 1. 헤더 좌측 (279px 필요)
- "Markdown Editor" 타이틀: ~110px
- gap-3: 12px
- "브라우저 로컬에 안전하게 저장됨": ~157px ← 가장 긴 텍스트

### 2. 헤더 우측 (274px 필요)
- 모바일 탭 전환 (편집/미리보기): ~120px
- 아이콘 버튼 5개 (28px × 5): ~140px
- 구분선 + 마진: ~10px

### 3. 프리뷰 영역
- 코드 블록: overflow-x: auto로 수평 스크롤 가능 (OK)
- 테이블: 3열 테이블이 모바일에서 잘릴 수 있음
- 긴 URL/인라인코드: word-break 미설정으로 넘침 가능

### 4. 하단 상태바
- "XX 줄 / XX 자 / XX 단어 / Markdown / UTF-8" 모두 한 줄에 표시
- 모바일에서 양쪽 justify-between으로 겹칠 수 있음

### 5. 에디터 textarea
- 모노스페이스 폰트 0.9rem: 모바일에서 적절
- word-wrap 기본 동작으로 줄바꿈은 됨
- 하지만 긴 코드 라인은 수평 스크롤 없이 잘림
